# Admin Demo v3

CreatiAds 权限管理后台（极简版）

## 📋 核心设计

### 两级固定角色

**企业角色（Organization Role）：**
- 👑 **Admin（管理员）**：可管理组织、邀请成员、自动拥有所有空间的最高权限
- 👤 **Member（成员）**：只能访问被邀请加入的空间

**空间角色（Workspace Role）：**
- 👑 **Owner**：空间管理者，可管理空间、邀请成员、修改空间设置
- ✏️ **Editor**：可创建/编辑空间内的资源（BI 报表、数据源等）
- 👁 **Viewer**：访客，只能查看别人分享给自己的资源

### 权限叠加规则

```
企业 Admin + 任意空间 = 自动拥有该空间的 Owner 权限（无需显式配置）
企业 Member + 空间 Owner = 在该空间内拥有管理权限
企业 Member + 空间 Editor = 在该空间内可编辑
企业 Member + 空间 Viewer = 在该空间内只能看分享的资源
企业 Member + 未加入空间 = 无法访问该空间
```

## 🎨 设计特点

1. **极简菜单**：只有 3 个菜单（成员管理、空间管理、订阅与计费）
2. **固定角色**：不再支持自定义角色，避免配置复杂
3. **直观展示**：成员列表直接显示加入的空间及角色
4. **智能权限**：企业 Admin 自动拥有所有空间权限，无需单独配置

## 🔄 与 v2 的对比

| 维度 | v2 | v3 |
|------|----|----|
| 菜单数量 | 6 个（成员/部门/角色/工作空间/授权/计费）| 3 个（成员/空间/计费）|
| 角色 | 可自定义角色 + 权限矩阵 | 固定 2+3 角色 |
| 部门 | 支持部门层级 | ❌ 无部门概念 |
| 工作空间 | 归属于部门 | 扁平结构，直接归属组织 |
| 权限模型 | 复杂（功能点 × 数据范围） | 简单（两级固定角色） |
| 适用场景 | 大型企业、复杂权限需求 | 中小团队、项目制协作 |

## 💡 使用建议

**适合 v3 的场景：**
- 广告代理公司（多客户、项目制）
- 中小团队（10-50 人）
- 不需要复杂部门管理
- 以"项目空间"为核心的协作

**适合 v2 的场景：**
- 大型企业（需要部门层级）
- 需要细粒度的功能点权限控制
- 需要自定义角色

## 🛠 技术栈

- 纯 HTML + CSS + JavaScript
- 无框架依赖
- 数据存储在内存（刷新丢失）
